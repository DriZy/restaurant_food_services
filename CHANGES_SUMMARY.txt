KEY FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
FEATURE 1: Force Plugin Authentication
├─ Redirects WooCommerce my-account to plugin signup
├─ Blocks wp-login.php access (existing functionality enhanced)
├─ Centralizes all auth flows through plugin
└─ Preserves redirect URLs for user experience
FEATURE 2: Authentication Guard
├─ Checks authentication on wizard submission
├─ Shows modal dialog if not authenticated
├─ Saves wizard state to localStorage
├─ Redirects to signup with state identifier
└─ Restores state after authentication
FEATURE 3: Wizard State Preservation
├─ Saves all user input to localStorage
├─ Survives page navigation
├─ Survives login/logout cycles
├─ Auto-restored with success notification
└─ Cleared after successful submission
FEATURE 4: Responsive Design
├─ Mobile-first approach
├─ Full support for 320px+ screens
├─ Optimized touch targets for mobile
├─ Tablet-specific layouts
├─ Desktop optimizations
└─ Print-friendly styles included
FEATURE 5: Authentication Modal
├─ Non-intrusive overlay
├─ Clear messaging
├─ Single CTA button
├─ Smooth animations
├─ Keyboard accessible
└─ Mobile optimized

1. ACTIVATE PLUGIN
   - Upload files to wp-content/plugins/
   - Activate from WordPress admin
2. CREATE SIGNUP PAGE
   - Add new WordPress page
   - Insert [restaurant_signup] shortcode
   - Publish
3. CONFIGURE SETTINGS
   - WooCommerce > Settings > Accounts
   - Enable "Guest Checkout": NO
   - Enable "Customer Registration": YES
4. TEST AUTHENTICATION FLOW
   - Log out of WordPress
   - Visit catering/meal plan form
   - Try to submit without logging in
   - Verify modal appears
   - Verify redirect to signup works
   - Verify state restoration after login

TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Problem: Modal doesn't appear
Solution:
  1. Check browser console for errors
  2. Verify auth-guard.js loaded (Network tab)
  3. Check RestaurantAuthGuard object exists
  4. Verify isUserAuthenticated() returns false
Problem: State not restoring
Solution:
  1. Verify localStorage is enabled in browser
  2. Check URL parameter restaurant_wizard_type present
  3. Check localStorage keys in DevTools
  4. Clear localStorage and try again
Problem: Responsive design not working
Solution:
  1. Hard refresh browser (Ctrl+Shift+R)
  2. Disable browser cache in DevTools
  3. Check for theme CSS conflicts
  4. Verify viewport meta tag in HTML head
Problem: Buttons not full-width on mobile
Solution:
  1. Check CSS file loaded correctly
  2. Verify no parent overflow-x: hidden
  3. Check theme container widths
  4. Test in incognito mode
